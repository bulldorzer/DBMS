----------<와인 추천 시스템 기획서 1차기획서 >-----------
1. root테이블
	- 와인 ID값으로 모든테이블을 참조가능하며 일반인 또는 소믈리에가 
	필요한 정보 (와인의 원산지, 이름, 빈티지, 맛, 페어링추천음식 등등)에 쉽게 접근할 수 있는 테이블
	즉 메인화면
		※ 관리정보
			- 와인 ID
			- 와인이름
			- (설계중 필요시 컬럼 추가 예정)

2. 와인 정보 (Wine_Information) 테이블
	- 와인의 정보를 관리 하는 테이블
		※ 관리정보
			1. 와인 ID
			2. 레드와인, 화이트와인 종류
				red wine
				white wine
				rose wine
				sparking
				blending wine
			3. 포도품종
			4. 빈티지
			5. 맛(전문소믈리에의 평균적인 평가) -코드처리 할것 -> 매핑테이블 추가할것
				s - sweet (단맛)
				d - dry (단맛없는)
				sk - sparkling (탄산함유)
				sm - smooth (부드러운)
				as - Astringency (떫은맛)
			7. 향	
			8. 생산국
			9. 와인 도수
			10. 와인 이름

3. 맛 코드 테이블
	- 와인의 맛을 코드화한 테이블 2번 테이블에서 참조하여 사용할 테이블
		※ 관리정보
			5. 맛(전문소믈리에의 평균적인 평가) -코드처리 할것 -> 매핑테이블 추가할것 
				s - sweet (단맛)
				d - dry (단맛없는)
				sk - sparkling (탄산함유)
				sm - smooth (부드러운)
				as - Astringency (떫은맛)

4. 와인 가게 (Wine_Store) 테이블
	- 해당하는 와인 ID의 판매중인 가게의 정보를 제공하는 테이블
		※ 관리정보
			1. 와인 ID
			2. 가게 ID
			3. 가게 이름
			4. 가게 주소
			5. 와인 가격
			6. 와인 수량

5. 와인 페어링(Wine_Pairing) 추천음식
	- 해당하는 와인과 페어링하기 좋은 음식 추천 정보를 제공하는 테이블
		※ 관리정보
			1. 와인 ID
			2. 페어링하기 좋은 맛
			3. 추천 음식

6. 아이디의 권한 분류 (user_pos)
	wine_main,mem_info와의 매핑 테이블 만들어 권한 분류가 필요함
		※ 관리정보
			1. 아이디
			2. 사용자 권한 (default user)
				admin - 관리자
				user - 사용자

7. 와인 추가 건의(Wine_add) 
	- 회원이 데이터의 없는 와인을 추가할때 건의하는 정보 테이블 
		※ 관리정보
			1. 와인 ID
			2. 페어링하기 좋은 맛
			3. 추천 음식
